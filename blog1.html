This first article is on the three tier architecture. We will call it TTA for convenience. So what's the TTA? Definition: A three-tier architecture is a client-server architecture in which the functional process logic, data access, computer data storage and user interface are developed and maintained as independent modules on separate platforms. This is the definition we get when we first Google the three tier architecture. So what is it?

So basically the three tier architecture works in this way:

1. There is a database
2. ‎There is a application server
3. ‎There is a client

So what happens behind the scene is

-->.
There is a web server like Apache, Nginx which can convert the server side code to client side code
-->.
‎Now why is it necessary? Because languages like PHP, asp.net are server side languages.
-->.
‎Now the question is what is a server side language and why is it necessary?
-->.
‎So basically the client (in most cases the web browser) understands only Markup languages (like Html) , css, JavaScript. Also for security reasons the web developer might not want to reveal the code, entire web logic etc.
-->.
‎So in a gist; HTML, CSS, JAVASCRIPT are fundamentally front end languages and are only needed to view in the browser or any graphical user interface. So next time when you think why do they show source code in browser isn't it a security confidentiality breach? Remember the browser source code is just the code which the browser used to understand what it has to show you in the GUI and may not in most cases reveal anything important about the architecture. Also JavaScript is basically a front end language ( although many server side language also use it in backend but that's a discussion for another day) so any cross site scripting you may achieve in a site may at maximum result in remote code execution on the client side and the server maybe totally be proofed from it.
-->.
‎Now the third part is the database. The database is where the data is stored by the SERVER. Now the web architecture should and in most cases works in a way where only an server can interact with the database. So basically any front end/ client side attacks can hardly harm the database.

So this is how the three tier architecture works. In future blogs I ll try to explain why this is necessary? So keep looking this space for more stuff. Until then goodbye :)
